## Experiment 3: Live Data

### Overview

This week I explored live data through a digital approach in p5.js. I wanted to create something that was not only responsive to changing information, but also visually expressive and connected to my own design interests. Since I often enjoy working with character-based graphics and visual branding, I developed a weather-responsive cat emotion system. The idea was to use live weather data to control the cat’s appearance and behaviour, so that the character became a playful visual interface for environmental conditions.

Rather than presenting weather as plain numbers or text, I wanted to translate it into mood, colour, and movement. This helped me think about how live data can become part of a design language and communicate information in a more emotional and engaging way.

### Activity 2: Developing a p5.js Weather Visualisation

For my independent study, I chose a digital approach and used p5.js to build a live data visualisation based on weather conditions. I developed the concept of a cat character whose emotional state changes according to real-time weather data. I selected weather as my data source because it is easy to relate to everyday life, changes constantly, and offers multiple variables that can be mapped to visual properties.

In my sketch, different weather values controlled different parts of the composition. Temperature influenced the overall colour of the cat.

 Higher temperatures shifted the cat toward warmer tones, while lower temperatures suggested cooler colours. Rain or cloudy conditions changed the emotional feeling of the design by making the cat appear more muted or blue-toned. Wind speed affected the movement of small elements such as shaking whiskers or subtle body motion, helping the character feel more alive. Humidity influenced visual softness, opacity, or blur-like effects to suggest a heavier atmospheric condition.

### Design Decisions and Mapping

One of the most important parts of this experiment was deciding how to map abstract data values into visual form. I wanted the system to be simple enough to read quickly, but expressive enough to make the weather feel atmospheric rather than technical.

I chose to use the cat as a visual metaphor because characters can communicate emotion very effectively. A slight change in colour, expression, or movement can suggest a different mood immediately. This made the design more engaging than a conventional chart. It also connected well to my interest in brand identity and graphic communication, because the outcome felt closer to a visual identity system than a standard data visualisation.

My mapping strategy translated environmental variables into behavioural visual parameters through the following structure:
Temperature controlled colour to represent thermal variation across environments.
Weather conditions influenced mood or facial expression to communicate atmospheric change.
Wind speed controlled movement to introduce responsiveness into the character interface.
Humidity affected visual softness or heaviness to suggest differences in environmental density.
This structure allowed environmental variables to function as behavioural inputs rather than static numerical values.
This approach positioned live data as an active design material rather than a passive source of information.
This approach allowed invisible environmental data to become perceptible through character behaviour rather than numerical representation.

### What the Work Reveals

This work communicates that weather is not just a set of measurements, but also something that affects feeling and atmosphere. By translating live weather data into the visual behaviour of a cat character, I was able to create a representation that feels more personal and emotionally readable. The design does not aim to provide scientific precision. Instead, it interprets data in a way that makes the information more immediate, memorable, and expressive.

I think this project reveals how live data can be used not only to inform, but also to create visual storytelling. The changing state of the cat reflects the changing state of the environment, making the data feel more alive.

### Process and Learning

This experiment helped me improve my understanding of how data can be connected to visual parameters in p5.js. I learned how live information can be fetched from an external source and then mapped to features such as colour, shape, and motion. I also learned that successful live data visualisation depends not only on technical connection, but also on thoughtful interpretation. The same data can be visualised in many different ways, so the design decision is just as important as the code.

I also found that testing different values was important. Small adjustments in scale, size, and movement made a big difference in whether the output felt readable or too chaotic. This reminded me that live data design requires both structure and sensitivity.


### Relation to Practitioner Examples

This experiment relates to the practitioner examples discussed in class because it transforms live data into a more experiential form. Rather than displaying weather as text alone, I translated it into a visual and emotional system. This connects to the idea that live data can become material for expression, not just information. Similar to how practitioners use motion, installation, or systems to reveal changing conditions, my project uses a character-based interface to communicate environmental change in a more playful and accessible way. This comparison helped me understand how live data visualisation can operate as an experiential interface rather than a purely descriptive display.

### What I Would Develop Further

With more time, I would like to make the cat system more refined and interactive. For example, I could add facial expressions, animated transitions between different weather states, or a more developed visual identity around the character. I would also like to experiment with using weather forecasts over time, so the sketch could show change across hours rather than only the current moment.

Another possible development would be turning the cat into a small branded interface or digital widget, where live data and character design are combined more clearly as a product-like experience. This would connect even more strongly to my interest in graphic design and visual branding.
