# Week 02
## Experiment 2: Interactivity

### In-Class Activities
### Overview
This week I explored how to create visual compositions using p5.js and experimented with DOM elements to introduce interaction. Through these activities, I learned how shapes, colour relationships, and user input can influence visual outcomes and make digital drawings more dynamic.

<img src="../assets/week-02/Week2 img4.png" width="800"> </center> <center><sub>

In this independent study task, I translated my hand-drawn data portrait from Week 01 into an interactive p5.js sketch. Instead of presenting the data as a fixed visual composition, I explored how interaction could allow viewers to experience variations in the data through colour and positional changes.

To achieve this, I reconstructed the structure of my original drawing using programmable shapes and experimented with arranging multiple elements across the canvas. I then adjusted colour combinations and spacing to test how different visual relationships affected readability and balance. This process helped me understand how translating analogue drawings into code requires simplifying visual components into repeatable systems.

Through this iteration, I also explored how interaction could make personal data more flexible and responsive. Compared with the original static version, the coded sketch allowed variation and experimentation, which supported a more exploratory way of presenting data.


### Reflection

Through these activities, I developed a better understanding of how programming can be used as a design tool rather than only a technical skill. I explored how shape placement, colour contrast, and interaction can influence visual communication. Compared with static drawings, interactive sketches allow users to participate in the visual experience. This process helped me see how p5.js can be used to create responsive and flexible visual compositions.
